Wacht ships identity, B2B access, API auth, webhooks, and an embedded AI runtime as one product layer.

The fastest way to build with it is to let your editor agent do it. **[Wacht Bench](/docs/guides/wacht-bench)** installs the skills, wires the Wacht Docs MCP, and gives the agent a typed CLI for projects, deployments, users, orgs, workspaces, and config — so Cursor, Claude Code, or Codex can ship Wacht features end-to-end.

## Start here

```bash
npx @wacht/bench init
```

That writes `AGENTS.md`, `.wacht/BOOTSTRAP.md`, and starter templates into the current project, installs the 14 Wacht skills under `./skills`, and prints the MCP config for your editor. Open the folder in your AI assistant and ask it to wire up auth.

To bootstrap a brand-new project:

```bash
npx @wacht/bench init --starter nextjs
npx @wacht/bench init --starter react-router
npx @wacht/bench init --starter tanstack
```

The full Bench guide is here: **[Wacht Bench](/docs/guides/wacht-bench)**.

## Core surfaces

### Console

The operator UI used to configure projects, deployments, authentication, billing, API authentication, webhooks, and AI features.

### Platform API

The Rust control plane, gateway, OAuth services, and worker stack used by operators and platform infrastructure.

### Frontend API

The customer-facing runtime API for sessions, organizations, workspaces, notifications, and end-user auth flows.

### SDKs

Frontend and server integrations layered on top of the runtime surfaces.
