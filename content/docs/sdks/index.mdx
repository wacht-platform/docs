Wacht SDKs are split into frontend adapters and server integrations.

## Decision diagram

```text
Start
 |
 +-- Frontend app?
 |     |
 |     +-- Next.js ----------> /sdks/nextjs
 |     +-- React Router -----> /sdks/react-router
 |     +-- TanStack Router --> /sdks/tanstack-router
 |     +-- Other React ------> /sdks/react (router-agnostic)
 |
 +-- Backend service?
       |
       +-- Node/TypeScript --> /sdks/node
       +-- Rust -------------> /sdks/rust/getting-started
```

## Pick your frontend path

- [Next.js](/sdks/nextjs) for App Router and middleware-driven protection.
- [React Router](/sdks/react-router) for loader/action protection in React Router apps.
- [TanStack Router](/sdks/tanstack-router) for route-tree and loader integrations.
- [React (router-agnostic)](/sdks/react) for plain React or custom routing stacks using `@wacht/jsx`.

## Pick your backend path

- [Backend JS SDK](/sdks/node) for Node/TypeScript services and server runtimes.
- [Rust SDK](/sdks/rust/getting-started) for typed async Rust backend integrations.
