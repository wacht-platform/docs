---
title: Installation
description: Install the TanStack Router SDK and configure the environment variables required for Wacht.
---

Install the package:

```bash
pnpm add @wacht/tanstack-router
```

Set the client-facing key in your app environment:

```bash
VITE_WACHT_PUBLISHABLE_KEY=pk_test_xxx
```

This is the only client-side variable the provider needs. The frontend host is encoded inside the publishable key, so you do not configure it separately. If you also call the backend SDK from a server runtime, set `WACHT_API_KEY` there too, but keep it out of the client bundle.

## Provider setup

Wrap your router provider:

```tsx
import { DeploymentProvider } from '@wacht/tanstack-router';

export function App() {
  return (
    <DeploymentProvider
      publicKey={import.meta.env.VITE_WACHT_PUBLISHABLE_KEY}
      uiOverrides={{
        app_name: 'Wacht',
      }}
    >
      <RouterProvider router={router} />
    </DeploymentProvider>
  );
}
```

`DeploymentProvider` loads the deployment for the current public key and makes the auth configuration available to the rest of the adapter.
