`@wacht/nextjs` is the Next.js integration for Wacht.

You will use two entrypoints:

- `@wacht/nextjs` for the provider, client-side hooks, and session-aware UI
- `@wacht/nextjs/server` for middleware, request auth, and the server client

The usual setup looks like this:

1. Mount `DeploymentProvider` once in your root layout.
2. Send signed-out users to the hosted auth pages.
3. Protect private routes in middleware.
4. Read auth state in server code where you need it.
5. Reuse the shared `Components` and `Hooks` pages when you are wiring UI and state together.
