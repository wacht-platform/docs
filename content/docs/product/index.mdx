Wacht is split across clear product surfaces:

- `console`: operator workflows and configuration
- `frontend-api`: runtime auth/session/user operations
- `platform-api`: control-plane and orchestration APIs
- SDKs: app-facing integrations over those APIs

## Surface diagram

```text
               | operators/config  |
               +---------+---------+

     +-----+-----------------------+-----+
     |             SDKs                  |
     | nextjs / react-router / tanstack  |
     | backend js / rust                 |
     +-----------------------------------+
```

## Common implementation shapes

- Frontend app with hosted auth and session state:
  Use [Next.js](/sdks/nextjs), [React Router](/sdks/react-router), or [TanStack Router](/sdks/tanstack-router).
- Backend service enforcing authenticated access:
  Use [Backend JS SDK](/sdks/node) or [Rust SDK](/sdks/rust/getting-started).
- B2B org/workspace workflows:
  Start with [Organizations and Workspaces](/sdks/rust/organizations-workspaces) and [B2B lifecycle guide](/guides/integration-playbooks/b2b-org-workspace-lifecycle).
